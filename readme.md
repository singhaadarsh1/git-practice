## 🌿 Branching Workflow Practice

To simulate real-world development, I avoid working directly on the main branch and follow a **feature-based workflow**.

### 🔄 Workflow

```bash
git switch -c feature-branch-name
# Make changes

git add .
git commit -m "describe changes"

git switch master
git merge feature-branch-name
git push
```

### 🎯 Why This Matters

* Encourages independent feature development
* Improves understanding of collaboration workflows
* Prepares for team-based projects
* Builds industry-level Git habits
